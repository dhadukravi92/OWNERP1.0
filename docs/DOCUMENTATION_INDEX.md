### 3. DESIGN_SYSTEM_IMPLEMENTATION.md {#implementation-guide}

**Length:** ~15,000 words  
**Audience:** Frontend developers, React component developers  
**Time to read:** 30-40 minutes  

**Contains:**
- **SECTION 1:** Complete CSS Custom Properties
  - Color tokens (primary, secondary, success, warning, danger, neutral)
  - Typography tokens (font-size, font-weight, line-height, letter-spacing)
  - Spacing tokens (Fibonacci grid: 4, 8, 12, 20, 32, 52, 84px)
  - Border-radius, shadows, transitions, semantic mappings

- **SECTION 2:** Component CSS Specifications
  - Buttons (sizing variants, color variants, states)
  - Cards (base, headers, bodies, aspect ratios, insight cards)
  - Forms (inputs, labels, validation states, error messages)
  - Tables (headers, rows, expandable rows, content)
  - Badges & Status indicators (colors, pulse animations)

- **SECTION 3:** React Component Examples
  - Button component (with ref forwarding)
  - Card components (Card, CardHeader, CardBody)
  - FormGroup component (label, error, hint handling)
  - ExpandableTableRow component (with state management)

- **SECTION 4:** Migration Checklist
  - Phase 1: Foundation (CSS tokens, component library setup)
  - Phase 2: Components (refactor existing buttons/cards/forms)
  - Phase 3: Pages (update each module)
  - Phase 4: Testing (visual regression, accessibility, performance)
  - Phase 5: Launch (documentation, deployment, monitoring)

**Key Takeaways:**
- Drop-in CSS variable system
- Production-ready React components
- Step-by-step migration checklist
- Cross-browser compatible

**When to use:**
- Implementing the design system
- Building new components
- Refactoring existing components
- Setting up CSS variables
- Creating component library
